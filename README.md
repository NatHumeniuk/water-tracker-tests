# WaterTracker App - Test Artifact Examples

## Project Overview

WaterTracker is a web application for tracking daily water consumption with user authentication and profile management. This repository demonstrates comprehensive QA testing skills through real-world test artifacts.

**Live Application:** [Water Tracker](https://watertracker-2o4z.onrender.com)  
**API Documentation:** [Swagger Docs](https://app.swaggerhub.com/apis-docs/NATGUMENUK/api-water-tracker/1.0.0)

---

## Testing Scope

**Backend API** - User registration, verification, sign in/out with JWT tokens, User profile updates and water consumption tracking.

**Frontend UI** - User interface components including header, navigation, and theme management.

## QA Skills Showcased

✅ **API Testing** - automated tests in Postman  
✅ **Test Case Design** - TestRail-format documentation with requirement traceability  
✅ **Bug Reporting** - Jira-format defect reports with severity analysis  
✅ **Requirement Analysis** - Linking test cases to functional requirements  
✅ **Negative Testing** - Edge cases, validation failures, security scenarios
